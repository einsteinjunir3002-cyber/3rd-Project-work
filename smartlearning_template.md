# CHAPTER ONE: INTRODUCTION
## 1.1 Background to the Study
Present a general background to the problem area... Explain the system that is currently used and its challenges, ending with the specific gap you are attempting to fill. Include a historical overview of the specific environment where the current problem is.

## 1.2 Statement of the Problem
Briefly describe the specific problem that the study engages in and the gap that the new project will resolve. Bullet points describing the problems of the previous/current system are ideal. Example includes:
- tedious process of records
- high turnaround and processing time
- loss of money
- loss of items
- difficulty in accessing information
- lack of communication
- poor storage or loss of files/data

## 1.3 Aim of the Study
State the general aim of the project.
Example: The aim of this study is to design and implement a [web/desktop/mobile] application for SmartLearning [context of the project environment].

## 1.4 Specific Objectives
State the specific objectives of the project. Example:
- To evaluate the current [context] existing system.
- To gather requirements for a new system.
- To design a new [context] system that improves on previous...
- To implement the system using appropriate tools/technologies.
- To test and evaluate the system to ensure that it meets the requirements.

## 1.5 Research Questions
These should align with the objectives. Example:
- What are the weaknesses in the current system?
- What are the requirements for a new system that will replace the existing system?
- How can a new system be modeled and implemented to solve the identified issues?
- How effective is the newly implemented system against the previous?
